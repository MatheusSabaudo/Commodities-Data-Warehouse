# Commodities Data Warehouse

## Project Overview

This project implements a **Data Warehouse for commodity trading analytics** using modern Data Engineering tools and best practices.

The pipeline ingests commodity price data and trading transactions, processes them through a layered transformation architecture, and produces analytics-ready datasets.

The project is built using **dbt (Data Build Tool)** to transform raw data into reliable, documented, and tested data models.

---

## Architecture

The data warehouse follows a **Medallion-style architecture**:

### Staging Layer
The staging layer cleans and standardizes raw source data.

Models:
- `stg_commodities`
- `stg_trading_data`

Responsibilities:
- Data type normalization
- Column renaming
- Standardized naming conventions
- Source data preparation

---

### Gold Layer

The gold layer contains **analytics-ready datasets** used for reporting and business insights.

Models:
- `dm_commodities`

Responsibilities:
- Join commodity prices with trading transactions
- Calculate trade values
- Compute earnings or losses
- Provide the latest trading snapshot

---

## Data Sources

The warehouse integrates the following sources:

| Source | Description |
|------|-------------|
| `commodities` | Historical commodity prices |
| `trading_data` | Trading transactions including buy/sell actions |

---

## Key Metrics

The project calculates several important metrics:

- **Price**  
  Total value of the trade  
  `price = quantity * closing_value`

- **Earnings**  
  Profit or loss generated by the trade  
  - Sell → Positive value  
  - Buy → Negative value

## Data Quality

The project includes built-in **data quality tests**:

- `not_null` constraints
- `accepted_values` validation for trading actions
- Column-level documentation

These tests ensure **reliable and trustworthy analytics datasets**.
